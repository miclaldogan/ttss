# Contributing Guide

Thank you for contributing to TTSS.

## Workflow

1. Fork the repository.
2. Clone your fork locally.
3. Create a feature branch from `main`.
4. Make focused commits with clear messages.
5. Run linting and tests locally.
6. Push the branch to your fork.
7. Open a pull request against `main`.

## Branch Naming

Use short descriptive branch names such as:

- `feature/temporal-labeling`
- `fix/inference-cli`
- `docs/readme-update`

## Pull Request Expectations

- Keep the scope of each PR narrow.
- Add or update tests when behavior changes.
- Update documentation for user-facing changes.
- Ensure CI passes before requesting review.

## Reporting Issues

When opening an issue, include:

- a concise description of the problem
- steps to reproduce
- expected behavior
- actual behavior
- environment details

## Code Style

- Follow PEP 8.
- Prefer explicit, readable implementations.
- Avoid unrelated refactors in the same PR.
